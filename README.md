## Prerequisites

- **LLM agent** вҖ” Claude or any agent that accepts skill/system prompts
- **Conda, Mamba, or Micromamba** вҖ” environment management (no bare pip installs)
- **Git** вҖ” version control initialized before running `02_root`
- **Python project** вҖ” any type: ML, ETL, data pipeline, tooling, web

---

## Quickstart

1. Clone this repo.
2. Copy the relevant skill file into your agent as a system prompt.
3. Point the agent at your project root.
4. The skill handles the rest via Q&A before any file is written.

Run skills in order for a new project.

---

## Core conventions

| Convention | Rule |
|---|---|
| **Append-only docs** | `docs/` files are ledgers вҖ” never overwritten, only appended |
| **Sandbox before production** | All agent code goes in `agent_sandbox/` first |
| **Conda/Mamba strictly** | No standard Python venvs or bare pip installs |
| **Q&A before any write** | Agent asks before touching any file |
| **Non-destructive guardrail** | Agent shows a diff and asks permission if user-written content would be touched |
| **B2 English output** | All documentation in clear, direct B2 English |
| **Sessions log** | One `sessions.md` entry per calendar day, append-only |
| **Skill chaining** | Each skill ends with a suggestion for the next logical skill |

## Ground truth files (generated by CRAFT)

Every project running CRAFT maintains these four files under `/docs/`:

- `tasks.md` вҖ” structured task blocks with status, stack, strategy, validation, hardware, risks, and known issues
- `knowledge_base.md` вҖ” categorized references: Literature & Theory / Libraries & Tools / Data & Formatting Specs / Git & Inspiration
- `sessions.md` вҖ” daily log: objective, decisions, next actions
- `data_lineage.md` вҖ” table tracking raw source вҶ’ processing script вҶ’ output for every data transformation
